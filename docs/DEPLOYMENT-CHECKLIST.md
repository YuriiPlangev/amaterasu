# Чеклист обновления — что и где обновить

## 1. Next.js (фронтенд)

**Что сделано в проекте:**
- Страница контактов (кнопки, компактность)
- Категории товаров (фильтр avatar, «Без категории»)
- Профиль (Admin-плашка, валидация displayName)
- Регистрация (валидация username, сохранение phone)
- Комментарии (Admin-плашка)
- И другие изменения

**Что нужно сделать:**
1. Выполните билд: `npm run build`
2. Задеплойте на хостинг (Vercel и т.п.) или перезапустите сервер

## 2. WordPress

### Файлы, которые должны быть в WordPress

| Файл | Куда добавить |
|------|----------------|
| `wordpress-custom-endpoint.php` | `functions.php` темы **или** отдельный плагин |
| `amaterasu-checkout.php` **или** `amaterasu-checkout-simple.php` | Отдельный плагин (в папке `wp-content/plugins/`) |
| `docs/wordpress-launch-day-avatars.php` | `functions.php` темы (если нужны аватарки за регистрацию) |

**Важно:** Должен быть активен только один плагин — либо `amaterasu-checkout`, либо `amaterasu-checkout-simple`. Оба регистрируют `custom/v1/register` и `custom/v1/login`. Рекомендуется **amaterasu-checkout.php** (полная версия с phone, display_name).

### Что добавлено в `wordpress-custom-endpoint.php`
- Поле `phone` в REST API пользователей
- Фильтр комментариев с полем `isAdmin`
- Запрет слова "Admin" в `display_name` при PATCH пользователя

### Что добавлено в `amaterasu-checkout-simple.php`
- Сохранение `phone` при регистрации
- Возврат `phone` и `display_name` при логине
- Валидация username (запрет "admin")

### Что добавлено в `amaterasu-checkout.php`
- Валидация username (запрет "admin")

## 3. Проверка после обновления

- [ ] Сайт открывается, нет ошибок в консоли
- [ ] Регистрация: ввод телефона → после входа телефон виден в профиле
- [ ] Профиль: сохранение displayName с кириллицей
- [ ] Профиль: при "Admin" в имени выводится понятная ошибка
- [ ] Регистрация: при "admin" в username выводится ошибка
- [ ] У аккаунта-админа в профиле показывается плашка "Admin"
- [ ] У админа в комментариях показывается плашка "Admin"
- [ ] В фильтрах каталога нет "avatar" и "Без категории"
- [ ] Кнопки соцсетей на странице контактов компактные

---

## 4. Если что-то не работает

1. **Данные не обновляются** — см. `docs/CACHING-TROUBLESHOOTING.md`
2. **Ошибки с кириллицей** — см. `docs/WP-RUSSIAN-DISPLAY-NAME.md`
3. **Телефон не сохраняется** — проверьте, что в WordPress активен `amaterasu-checkout.php` (или simple с правками), а не другой плагин с тем же маршрутом.
