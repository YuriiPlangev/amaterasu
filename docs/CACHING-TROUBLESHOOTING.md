# Почему не появляются изменения (новости, категории) — решение

## Где стоит кэш в проекте

| Что              | Где                  | Время кэша |
|------------------|----------------------|------------|
| Категории/фильтры| `/api/catalog-init`  | 1 час      |
| Категории        | `/api/catalog/filters` | 1 час   |
| Новости          | `/api/news`          | 1 мин      |
| Товары           | `/api/products`      | 1 мин      |
| React Query (клиент) | `useCatalogMetadata` | 1 час  |

## Возможные причины задержки (6+ часов)

### 1. Кэш на хостинге / CDN
- **Vercel** — кэширует ответы API. После `revalidate` следующий запрос должен обновить данные.
- **Cloudflare** или другой CDN — может кэшировать API-ответы дольше. Проверьте настройки кэша.
- **Решения:** Очистить кэш в панели хостинга или CDN, либо временно отключить кэш для проверки.

### 2. Кэш WordPress
- Плагины кэша (WP Super Cache, W3 Total Cache, WP Rocket и т.п.) могут кэшировать REST API.
- **Решение:** Отключить кэширование REST API в настройках плагина или временно отключить плагин.

### 3. Кэш в браузере
- React Query хранит данные в памяти: пока открыта вкладка, повторные запросы могут не выполняться.
- **Решение:** Полное обновление (Ctrl+F5) или режим инкогнито.

### 4. Нужен новый деплой
- Если используется полностью статический экспорт (`next export`), данные собираются только при билде.
- **Решение:** Сделать новый билд и деплой.

## Быстрые шаги для проверки

1. Открыть каталог/новости в **режиме инкогнито** (Ctrl+Shift+N).
2. Убедиться, что WordPress REST API отдаёт новые данные:
   - Новости: `https://ваш-wp.com/wp-json/wp/v2/posts?per_page=5`
   - Категории: через ваш `amaterasu/v1/catalog-init` или WooCommerce API.
3. Понизить время кэша для тестов (см. ниже).

## Как уменьшить кэш для тестирования

В `app/api/catalog-init/route.ts` и `app/api/catalog/filters/route.ts`:
- `revalidate: 3600` → `revalidate: 60` (1 мин).

В `hooks/useCatalogMetadata.ts`:
- `staleTime: 3600 * 1000` → `staleTime: 60 * 1000` (1 мин).

После проверки лучше вернуть прежние значения для производительности.
